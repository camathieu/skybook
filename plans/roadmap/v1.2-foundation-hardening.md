# v1.2 — Foundation Hardening

> Eliminate architectural debt identified during v1.1 closure review. Harden the foundation before building v2+ features (buddies, documents, BASE, tunnel).

A hardening milestone focused on data integrity, test coverage, and code hygiene. No new user-facing features — only correctness, safety, and future-proofing.

## Epics

- [Data Integrity](v1.2-foundation-hardening/01-data-integrity.md) — Fix non-atomic operations ands type inconsistencies
- [Code Quality](v1.2-foundation-hardening/02-code-quality.md) — Test coverage, DRY violations, API hygiene

## Goals

- Every mutation that changes multiple fields/positions is fully atomic
- `DateOnly` type has comprehensive unit tests
- No duplicate business logic between handler and metadata layers
- Type system is consistent (no `time.Time` ↔ `DateOnly` mismatches)
